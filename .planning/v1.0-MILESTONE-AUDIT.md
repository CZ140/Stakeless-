## Executive Summary

The v1.0 milestone has **one structural blocker**: Phase 04 (Game Infrastructure & Roulette) shipped without a `VERIFICATION.md`. This causes 11 requirements (GINF-06..09, ROUL-01..07) to be **orphaned** — absent from all phase VERIFICATION.md files, treated as unsatisfied. **Gap closure is narrow:** run Phase 04 verification, fix the Blackjack Double Down endpoint, and optionally address the `/profile` link. Everything else is SUMMARY frontmatter documentation debt.

## Integration Check Results

### Critical Code Defect

#### Blackjack Double Down — `dealerValue` missing from API response

- **Severity:** Critical
- **File:** `apps/backend/src/routes/games.ts` lines 758–764 (bust path) and 785–791 (normal path)
- **Root cause:** Both `res.json` calls in the `/blackjack/double` handler omit `dealerValue`. `BlackjackPage.tsx:403` destructures it as `undefined`. `revealDealerHand(dHand, undefined, result)` stores `undefined` in Zustand. Dealer hand total badge shows `"?"` permanently after any Double Down round — the number never resolves.
- **Fix:** Add `dealerValue: calculateHandValue(state.dealerHand).value` to both `res.json` calls in the double handler — same field already present in the `/blackjack/stand` response.
- **Requirements affected:** BJK-02, BJK-04

### Moderate Issue

#### Header `/profile` link — broken route

- **File:** `apps/frontend/src/components/Header.tsx:30`
- **Root cause:** `<Link to="/profile">` wraps `BalanceDisplay`. No `/profile` route in `App.tsx`. React Router wildcard (`path="*"`) silently redirects to `/`.
- **Note:** A profile *page* (PROF-01..03) is Phase 7 scope, not v1.0. AUTH-05 is about data storage — satisfied. But the active broken link needs a fix now: remove the Link wrapper or add a stub `ProfilePage` that states "Coming in a future update."
- **Requirements affected:** AUTH-05 (indirectly)

### Minor Issue

#### Roulette `selectedChip` not restored from localStorage

- **File:** `apps/frontend/src/stores/rouletteStore.ts:49`
- **Root cause:** `selectedChip: 10` hardcoded in initial state. Plinko/Mines/Blackjack stores seed `betAmount` from `localStorage` on init; Roulette does not.
- **Requirements affected:** GINF-07 (partial for Roulette; other 3 games correct)

#### Mute localStorage key inconsistency

- `rouletteStore`: key `'mute'` | Plinko: `'isMuted_plinko'` | Mines: `'isMuted_mines'` | Blackjack: `'isMuted_blackjack'`
- Per-game mute rather than global. Likely intentional; inconsistent naming is minor.
- **Requirements affected:** GINF-08 (met per-game; global mute not required by spec)

#### `POST /api/wallet/bet` — no frontend consumer

- Demo coin-flip backend route from Phase 3 pipeline build. No frontend page calls it. Dead route, harmless.
- **Requirements affected:** None
