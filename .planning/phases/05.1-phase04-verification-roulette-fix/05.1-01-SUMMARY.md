## Decisions Made

- Used `selectedChip_roulette` as the localStorage key (not `lastBet_roulette`) because chip denomination (10/50/100/500/Max) is distinct from a raw bet amount — the `lastBet_{game}` pattern in Plinko/Mines/Blackjack stores a raw coin amount, not a chip selection
- Normalized the Roulette mute key from `'mute'` to `'isMuted_roulette'` to match the `isMuted_{game}` convention already in place for plinkoStore, minesStore, and blackjackStore
- Did not implement a localStorage migration from `'mute'` to `'isMuted_roulette'` — accepted the one-time mute preference reset as a trivial v1 UX tradeoff per CONTEXT.md locked decisions
- VERIFICATION.md written after Task 1 code fix to ensure cited line numbers reflect the actual post-fix file state (avoiding Pitfall 3 from RESEARCH.md)
